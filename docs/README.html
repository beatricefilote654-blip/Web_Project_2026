UnD – Unemployment Data Visualizer

Instrument Web de vizualizare și comparare multi-criterială a datelor publice privind șomajul înregistrat din România, bazat pe statisticile oficiale ANOFM disponibile pe portalul date.gov.ro.
Disciplina: Tehnologii Web 2026 — Facultatea de Informatică, UAIC Iași
Versiune: 2.0 — Data: 2026-06-10

Rezumat

UnD este o aplicație Web server-side care permite vizualizarea și compararea multi-criterială a datelor publice privind șomajul înregistrat din România pe o perioadă de 63 de luni (ianuarie 2020 – aprilie 2025), adică L = 63 > 16.

Utilizatorii pot filtra datele după județ (toate cele 42 de județe ale României), grupă de vârstă, nivel de educație, mediu (urban/rural) și interval de timp, obținând vizualizări interactive (grafic cu bare, grafic liniar, grafic donut și hartă coropleth) exportabile în formatele CSV, JSON, SVG și PDF. Un modul de administrare separat permite importul manual și actualizarea automată a datelor direct din API-ul public al portalului data.gov.ro.

Cerințe funcționale implementate

Vizualizare (4 tipuri de grafice)

  • Grafic cu bare — rata medie a șomajului per județ în intervalul selectat
  • Grafic liniar — evoluția temporală a ratei medii lunare
  • Grafic donut (plăcintă) — distribuția pe categorii (vârstă / educație / mediu), selectabilă din UI
  • Hartă coropleth — rata șomajului suprapusă pe harta administrativă a României (SVG pur, fără tile-uri externe, proiecție D3 geoMercator)

Toate graficele sunt construite cu D3.js v7 și redau SVG-uri exportabile direct.

Filtrare multi-criterială

  • Județ: dropdown personalizat cu selecție multiplă, căutare live și butoane „Toate / Niciuna"
  • Grupă de vârstă: 15–24, 25–34, 35–44, 45–54, 55–64 (pill-checkboxes)
  • Nivel de educație: primar, secundar, superior
  • Mediu: urban, rural
  • Interval de timp: lună/an de start și lună/an de final (2020–2025)

Export date

  • CSV — descărcare directă din api/export.php?format=csv
  • JSON — descărcare directă din api/export.php?format=json
  • SVG — serializare D3 pentru fiecare grafic: bare, linie, donut, hartă (4 butoane)
  • PDF — via window.print() cu CSS @media print adaptat

Administrare

  • Autentificare cu sesiune PHP (parole stocate bcrypt)
  • Dashboard cu statistici: număr înregistrări, luni de date, intrări în cache
  • Import date manual: fișier CSV sau JSON
  • Actualizare date din API — buton care apelează api/admin.php?action=refresh_data; descarcă și parsează automat toate lunile disponibile pe data.gov.ro prin Server-Sent Events (progres în timp real)
  • Golire cache interogări (TTL 3600s)
  • Navigare și ștergere înregistrări (paginat, 50/pagină)

Datele reale provin de la ANOFM prin API-ul CKAN al portalului data.gov.ro și acoperă 63 de luni (ianuarie 2020 – aprilie 2025), 42 de județe, 5 grupe de vârstă, 3 niveluri de educație, 2 medii = 26.708 înregistrări.

Securitate

  • Injecție SQL: toate interogările folosesc PDO::prepare() cu parametri legați (execute([$p1, $p2, …])). Nicio valoare provenită din input utilizator nu este interpolată direct în SQL.
  • XSS: valorile afișate în template-uri JS folosesc un helper esc(s) bazat pe String.replace pentru caracterele & < > "; output-ul API PHP este JSON pur (nu HTML).
  • Autentificare: parole stocate cu password_hash(PASSWORD_BCRYPT), verificate cu password_verify(). Sesiunile PHP protejează toate acțiunile admin.
  • Protecție fișiere: .htaccess blochează accesul direct la directoarele db/ și scripts/.
  • Headers de securitate: X-Content-Type-Options: nosniff, X-Frame-Options: SAMEORIGIN, X-XSS-Protection: 1; mode=block.

Mecanism de caching

Fiecare cerere către api/data.php generează un cache_key = md5(json_encode(filters)). Rezultatul JSON este stocat în tabela cache_entries cu un TTL de 3600 secunde. La fiecare cerere sistemul verifică mai întâi cache-ul; dacă intrarea există și nu a expirat, returnează payload-ul direct fără a interoga tabela principală — reducând semnificativ latența pentru filtre repetate.

Există două niveluri de invalidare:

  • Golire cache interogări — șterge doar cache_entries; datele rămân intacte, interogările se re-calculează la prima accesare.
  • Actualizare date din API — șterge atât cache_entries cât și unemployment_records, apoi repopulează din data.gov.ro.

Design responsiv

Interfața utilizează CSS Grid și Flexbox fără niciun framework CSS extern, cu breakpoint-uri la 960px și 640px:

  • Sub 960px: graficele trec de la 2 coloane la 1 coloană.
  • Sub 640px: header-ul se stivuiește, subtitlul dispare pentru economie de spațiu, fiecare grup de filtre ocupă lățimea completă, butoanele de export se micșorează și se împachetează pe mai multe rânduri, înălțimea graficelor scade la 240px, iar harta la 300px.

Butoanele și țintele de atingere au un minim de 32px înălțime; dropdown-ul de județe folosește min(240px, 100vw - 1.5rem) pentru a nu depăși ecranul.
